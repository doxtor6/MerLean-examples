22 Def 10: Equivalence Relations in the Gauged Code

This chapter defines three fundamental equivalence relations on operators in the gauged code:

  1. Decorated stabilizer equivalence \(\sim _{\mathfrak {s}}\)

  2. Vertex stabilizer equivalence \(\sim _{\mathfrak {A}}\)

  3. Flux stabilizer equivalence \(\sim _{\mathfrak {B}}\)

  4. Full gauged code equivalence \(\sim _g\) (combining all three)

22.1 Operators on the Gauged System

\[ \mathsf{GaugedOperator}(w, V, E, M) := \mathsf{GaugedSystemState}(w, V, E, M) \to \mathsf{GaugedSystemState}(w, V, E, M) \]

Theorem 22.6 Operator Power Addition
✓
#

Uses

  • Definition 22.2
  • Definition 22.2

Lean declarations

  • GaugingLDPC.operatorPowNat_add

For any operator \(\mathsf{op}\) and natural numbers \(m, n\), we have:

\[ \mathsf{op}^{m+n} = \mathsf{op}^m \circ \mathsf{op}^n \]
Proof ▶

We proceed by induction on \(m\).

Base case (\(m = 0\)): By simplification using the definition, \(\mathsf{op}^{0+n} = \mathsf{op}^n = \mathrm{id} \circ \mathsf{op}^n = \mathsf{op}^0 \circ \mathsf{op}^n\).

Inductive step: Assume \(\mathsf{op}^{m+n} = \mathsf{op}^m \circ \mathsf{op}^n\). Then:

\[ \mathsf{op}^{(m+1)+n} = \mathsf{op}^{(m+n)+1} = \mathsf{op} \circ \mathsf{op}^{m+n} = \mathsf{op} \circ (\mathsf{op}^m \circ \mathsf{op}^n) = (\mathsf{op} \circ \mathsf{op}^m) \circ \mathsf{op}^n = \mathsf{op}^{m+1} \circ \mathsf{op}^n \]

by the inductive hypothesis and associativity of composition.

22.4 Decorated Stabilizer Equivalence \(\sim _{\mathfrak {s}}\)

Definition 22.11 Decorated Stabilizer Step
✓
#

Uses

  • Definition 22.1
  • Definition 14.8

Lean declarations

  • GaugingLDPC.DecoratedStabilizerStep

The one-step relation for decorated stabilizers is defined as: \(O \to _1 O'\) if and only if there exists \(s \in S\) such that

\[ O' = O \circ \mathbb {S}_s \quad \text{or} \quad O = O' \circ \mathbb {S}_s \]

where \(\mathbb {S}_s = \mathsf{decoratedStabilizer}(\mathsf{stabilizers}(s))\).

Definition 22.12 Decorated Stabilizer Equivalence
✓
#

Uses

  • Definition 22.11

Lean declarations

  • GaugingLDPC.DecoratedStabilizerEquiv

Two operators \(O\) and \(O'\) are decorated stabilizer equivalent, written \(O \sim _{\mathfrak {s}} O'\), if they are related by the equivalence closure of the one-step relation. That is, there exists a chain:

\[ O = O_0 \to O_1 \to \cdots \to O_n = O' \]

where each step applies (or removes) one decorated stabilizer.

22.7 Full Gauged Code Equivalence \(\sim _g\)

Definition 22.30 Gauged Code Step
✓
#

Uses

  • Definition 22.11
  • Definition 22.18
  • Definition 22.24

Lean declarations

  • GaugingLDPC.GaugedCodeStep

The combined one-step relation is defined as the disjunction of all three step relations:

\[ O \to _1 O' \iff O \to _{\mathfrak {s}} O' \lor O \to _{\mathfrak {A}} O' \lor O \to _{\mathfrak {B}} O' \]

That is, \(O' = O \circ S\) for any stabilizer \(S\) (decorated, vertex, or flux).

Definition 22.31 Gauged Code Equivalence
✓
#

Uses

  • Definition 22.30

Lean declarations

  • GaugingLDPC.GaugedCodeEquiv

Two operators \(O\) and \(O'\) are gauged code equivalent, written \(O \sim _g O'\), if they are related by the equivalence closure of the combined one-step relation. This captures:

\[ O \sim _g O' \iff \exists \text{ chain } O = O_0 \sim O_1 \sim \cdots \sim O_n = O' \]

where each \(\sim _i\) is one of \(\sim _{\mathfrak {s}}\), \(\sim _{\mathfrak {A}}\), or \(\sim _{\mathfrak {B}}\).

Theorem 22.39 Decorated Stabilizer Equivalence Implies Gauged Code Equivalence
✓
#

Uses

  • Definition 22.12
  • Definition 22.31
  • Definition 22.12
  • Definition 22.31

Lean declarations

  • GaugingLDPC.decoratedStabilizerEquiv_subset_gaugedCodeEquiv

If \(O \sim _{\mathfrak {s}} O'\), then \(O \sim _g O'\).

Proof ▶

Let \(h : O \sim _{\mathfrak {s}} O'\). We proceed by induction on the equivalence closure.

Case rel: If \(x\) and \(y\) are related by the one-step relation, then by applying the left injection to the gauged code step relation, we obtain \(x \sim _g y\).

Case refl: For any \(x\), we have \(x \sim _g x\) by reflexivity.

Case symm: If \(x \sim _g y\) by the inductive hypothesis, then \(y \sim _g x\) by symmetry.

Case trans: If \(x \sim _g y\) and \(y \sim _g z\) by the inductive hypotheses, then \(x \sim _g z\) by transitivity.
