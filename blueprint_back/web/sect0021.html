21 Lem 7: Gauged Code is Sparse and Distance Preserving

This chapter establishes that if the gauging graph \(\mathcal{G}\) satisfies all four criteria in Definition 9, then the gauged code is:

  1. Sparse (LDPC): All stabilizer generators (\(\mathbb {A}_v\), \(\mathbb {B}_p\), \(\mathbb {S}\)) have weight \(O(1)\).

  2. Distance-preserving: The code distance is preserved under gauging (\(d_{\text{gauged}} \geq d_{\text{original}}\)).

21.1 LDPC Property of Original Code

Definition 21.1 LDPC Code
✓
#

Uses

  • Definition 14.8

Lean declarations

  • GaugingLDPC.IsLDPCCode

An LDPC code has bounded stabilizer weight. Given a matter assignment \(\sigma \) and bounds matterWeightBound and stabilizerSupportBound, the code is LDPC if:

  1. Each vertex set \(\Sigma (v)\) has bounded size: for all \(v \in V\), \(|\{ m \in M : m \in \sigma .\Sigma (v)\} | \leq \texttt{matterWeightBound}\).

  2. The original stabilizers have \(O(1)\) support: \(\texttt{stabilizerSupportBound} \leq \texttt{matterWeightBound}\).

21.2 Part 1: Sparsity of the Gauged Code

Definition 21.2 Vertex Stabilizer Weight
✓
#

Uses

  • Definition 14.8

Lean declarations

  • GaugingLDPC.vertexStabilizerWeight

The total support weight of vertex stabilizer \(\mathbb {A}_v\) is:

\[ \texttt{vertexStabilizerWeight}(G, \sigma , v) = |\Sigma (v)| + |\text{incidentEdges}(v)| \]

where \(|\Sigma (v)|\) is the number of matter qudits associated with vertex \(v\), and \(|\text{incidentEdges}(v)|\) is the number of edges incident to \(v\).

Definition 21.4 Decorated Stabilizer Total Weight
✓
#

Uses

The total weight of decorated stabilizer \(\mathbb {S}\) is the sum of matter support and edge support:

\[ \texttt{decoratedStabilizerTotalWeight}(\text{matterSupport}, \text{edgeSupport}) = \text{matterSupport} + \text{edgeSupport} \]

where the matter support comes from the original stabilizer and the edge support from the path \(\alpha \) solving \(\partial _1 \alpha = \chi \).

Theorem 21.5 Incident Edges Card Equals Degree
✓
#

Uses

  • Definition 14.8
  • Definition 14.8

Lean declarations

  • GaugingLDPC.incidentEdges_card_eq_degree

The number of incident edges equals the graph degree for simple graphs. For a gauged cell complex \(G\) and vertex \(v\), given an equivalence between edges incident to \(v\) and the neighbor finset:

\[ |\text{incidentEdges}(G, v)| = \deg _G(v) \]
Proof ▶

We first establish that the cardinality of the subtype \(\{ e : E \mid \text{source}(e) = v \lor \text{target}(e) = v\} \) equals the cardinality of the neighbor finset, using the given equivalence. Unfolding the definitions of incidentEdges and SimpleGraph.degree, and using the cardinality filter property, we convert the statement to show equality of cardinalities of subtypes. By simplification of the filter and subtype cardinalities, the result follows.

Theorem 21.6 Gauged Code is Sparse
✓
#

Uses

  • Definition 16.20
  • Definition 14.8
  • Definition 16.20
  • Definition 14.8

Lean declarations

  • GaugingLDPC.GaugedCodeIsSparse

If the gauging graph satisfies all four criteria (Def 9) and the original code is LDPC, then all stabilizer generators have \(O(1)\) weight:

  • \(\mathbb {A}_v\): \(|\Sigma (v)| + \deg (v) = O(1)\)

    • \(|\Sigma (v)| = O(1)\) by LDPC property of original code

    • \(\deg (v) \leq \texttt{degreeBound} = O(1)\) by Criterion 1

  • \(\mathbb {B}_p\): \(|\partial p| \leq \texttt{cycleWeightBound} = O(1)\) by Criterion 2

  • \(\mathbb {S}\): matter support + edge support \(= O(1)\)

    • Matter support \(= O(1)\) by LDPC property

    • Edge support \(\leq \texttt{pathWeightBound} = O(1)\) by Criterion 3

Formally: Given a valid gauging graph \(VG\) with LDPC code property, we have:

  1. For all \(v \in V\): \(\texttt{vertexStabilizerWeight}(G, \sigma , v) \leq \texttt{matterBound} + VG.\texttt{degreeBound}\)

  2. For all \(p \in P\): \(\texttt{plaquetteStabilizerWeight}(G, p) \leq VG.\texttt{cycleWeightBound}\)

  3. For all \(S \in \texttt{stabilizers}\) and \(\chi \in S.\texttt{chargeConfigs}\), there exists \(\alpha : E \to _0 \mathbb {Z}/w\mathbb {Z}\) such that \(\alpha \) is a neutralizing path for \(\chi \) with \(\texttt{chainWeight}(\alpha ) \leq VG.\texttt{pathWeightBound}\)

Proof ▶

We prove the three parts separately.

Part 1 (\(\mathbb {A}_v\) weight bound): Let \(v\) be an arbitrary vertex. Unfolding the definition of vertexStabilizerWeight, the weight is \(|\Sigma (v)| + |\text{incidentEdges}(v)|\). By the LDPC property, \(|\Sigma (v)| \leq \texttt{matterBound}\). By the incident bound hypothesis, \(|\text{incidentEdges}(v)| \leq \deg (v)\). By Criterion 1 (bounded degree property), \(\deg (v) \leq VG.\texttt{degreeBound}\). By linear arithmetic (omega), the sum is at most \(\texttt{matterBound} + VG.\texttt{degreeBound}\).

Part 2 (\(\mathbb {B}_p\) weight bound): For any plaquette \(p\), the plaquette stabilizer weight is bounded by \(VG.\texttt{cycleWeightBound}\) directly from Criterion 2 (sparse cycles property).

Part 3 (\(\mathbb {S}\) edge weight bound): For any stabilizer \(S\) in the stabilizer set and any charge configuration \(\chi \) in \(S\)’s charge configs, Criterion 3 (constant path property) directly provides the existence of a neutralizing path \(\alpha \) with the required weight bound.

Theorem 21.9 Weight Non-Decrease via Lemma 6
✓
#

Uses

  • Definition 14.8
  • Definition 16.20
  • Definition 14.8

Lean declarations

  • GaugingLDPC.weight_non_decrease_via_Lem6

For any operator \(O\) with vertex support \(V_O\) and edge support \(E_O\), where \(U \subseteq V_O\) is the subset being “traded” for edge support, the new operator \(O' = (\prod _{v \in U} \mathbb {A}_v) \cdot O\) has:

  • Vertex support \(V_O \setminus U\)

  • Edge support \(E_O \cup \partial U\)

By Lemma 6’s boundary_ge_size_for_expander: \(|\partial U| \geq |U|\).

Therefore:

\[ |V_{O'}| + |E_{O'}| = (|V_O| - |U|) + (|E_O| + |\partial U|) \geq |V_O| + |E_O| \]

Formally: For an expander graph \(G\) and small subset \(U\):

\[ |\text{edgeBoundaryFinset}(G, U)| \geq |U| \]
Proof ▶

This follows directly by applying boundary_ge_size_for_expander from Lemma 6.

Theorem 21.10 Weight Change Under Stabilizer Equivalence
✓
#

Uses

  • Definition 16.20
  • Definition 14.8
  • Definition 16.20
  • Definition 14.8

Lean declarations

  • GaugingLDPC.weight_change_under_stabilizer_equiv

When multiplying an operator by \(\prod _{v \in U} \mathbb {A}_v\):

  • Matter support decreases by at most \(|U|\)

  • Edge support increases by at least \(|\partial U|\)

For an expander graph with \(h(\mathcal{G}) \geq 1\), the Cheeger bound gives \(|\partial U| \geq |U|\), so total weight does not decrease.

Formally: Given an expander graph \(G\), small subset \(U\), original weight, matter decrease, and edge increase satisfying \(\texttt{matterDecrease} \leq |U|\) and \(\texttt{edgeIncrease} \geq |\text{edgeBoundaryFinset}(G, U)|\):

\[ \texttt{originalWeight} - \texttt{matterDecrease} + \texttt{edgeIncrease} \geq \texttt{originalWeight} \]
Proof ▶

By the Cheeger bound (boundary_ge_size_for_expander), we have \(|\partial U| \geq |U|\).

We show \(\texttt{edgeIncrease} \geq \texttt{matterDecrease}\) by calculation:

\begin{align*} \texttt{edgeIncrease} & \geq |\text{edgeBoundaryFinset}(G, U)| & \text{(by hypothesis)} \\ & \geq |U| & \text{(by Cheeger bound)} \\ & \geq \texttt{matterDecrease} & \text{(by hypothesis)} \end{align*}

The final inequality \(\texttt{originalWeight} - \texttt{matterDecrease} + \texttt{edgeIncrease} \geq \texttt{originalWeight}\) follows by linear arithmetic (omega).

Theorem 21.11 Gauged Code Distance Preserved
✓
#

Uses

  • Definition 16.20
  • Definition 14.8
  • Definition 16.20
  • Definition 14.8

Lean declarations

  • GaugingLDPC.GaugedCodeDistancePreserved

For a gauging graph satisfying Criterion 4 (\(h(\mathcal{G}) \geq 1\)), the distance of the gauged code satisfies \(d_{\text{gauged}} \geq d_{\text{original}}\).

Theorem: Let \(L\) be any logical operator in the original code with weight \(w_L \geq d_{\text{original}}\). Let \(D = \mathfrak {D}(L)\) be the decorated operator with weight \(w_D = w_L + (\text{decoration edge weight})\). For any operator \(O\) in the equivalence class of \(D\) obtained by multiplying by \(\prod _{v \in U} \mathbb {A}_v\) (for any small subset \(U\)), we have \(|\text{Supp}(O)| \geq d_{\text{original}}\).

Proof outline:

  1. Decoration only adds edge support: \(w_D \geq w_L \geq d_{\text{original}}\)

  2. Stabilizer multiplication: By Lemma 6 and Criterion 4, when multiplying by \(\mathbb {A}_U\):

    • Matter lost \(\leq |U|\)

    • Edges gained \(\geq |\partial U| \geq |U|\) (by Cheeger bound \(h(\mathcal{G}) \geq 1\))

    • Therefore weight does not decrease

  3. Combining: All operators in the equivalence class have weight \(\geq d_{\text{original}}\)

  4. Therefore: \(d_{\text{gauged}} = \min _L \min _{O \sim _g \mathfrak {D}(L)} |\text{Supp}(O)| \geq d_{\text{original}}\)

Proof ▶

Let \(U\) be a small subset with matter decrease and edge increase satisfying the hypotheses. The decorated weight satisfies \(\texttt{logicalWeight} + \texttt{decorationWeight} \geq d_{\text{original}}\) by linear arithmetic since \(\texttt{logicalWeight} \geq d_{\text{original}}\).

By weight_change_under_stabilizer_equiv applied to the simple graph of \(G\), Criterion 4 (expansion property), \(U\), the smallness hypothesis, the decorated weight, matter decrease, and edge increase with the given bounds, we obtain that the weight after stabilizer multiplication is at least the decorated weight.

By linear arithmetic (omega), the final weight is at least \(d_{\text{original}}\).

Corollary 21.12 Gauged Distance \(\geq \) Original Distance
✓
#

Uses

  • Definition 16.20
  • Definition 14.8
  • Definition 16.20
  • Definition 14.8

Lean declarations

  • GaugingLDPC.gaugedDistance_ge_originalDistance

The gauged code distance is at least the original code distance:

\[ d_{\text{gauged}} \geq d_{\text{original}} \]

This combines decoration and stabilizer equivalence to show that every operator in the equivalence class of any decorated logical operator has weight \(\geq d_{\text{original}}\), hence \(d_{\text{gauged}} = \min \) over all such operators \(\geq d_{\text{original}}\).

The key insight is that for any small subset \(U\) used in stabilizer equivalence, the weight change is non-negative (by Lemma 6 and Criterion 4), so all operators in the equivalence class have weight \(\geq \) the original logical operator weight \(\geq d_{\text{original}}\).

Proof ▶

We extract the witness: there exist \(\texttt{logicalWeight}\), \(\texttt{decorationWeight}\), \(U\), \(\texttt{matterDecrease}\), and \(\texttt{edgeIncrease}\) such that \(\texttt{logicalWeight} \geq d_{\text{original}}\), \(U\) is a small subset, \(\texttt{matterDecrease} \leq |U|\), \(\texttt{edgeIncrease} \geq |\partial U|\), and \(d_{\text{gauged}} = (\texttt{logicalWeight} + \texttt{decorationWeight}) - \texttt{matterDecrease} + \texttt{edgeIncrease}\).

Applying GaugedCodeDistancePreserved with these parameters, we obtain that the right-hand side is at least \(d_{\text{original}}\).

By linear arithmetic (omega), \(d_{\text{gauged}} \geq d_{\text{original}}\).

Lean declarations

  • GaugingLDPC.GaugedCodeIsSparseAndDistancePreserving

If the gauging graph \(\mathcal{G}\) satisfies all four criteria in Definition 9, then:

  1. Sparse: All stabilizer generators (\(\mathbb {A}_v\), \(\mathbb {B}_p\), \(\mathbb {S}\)) have weight \(O(1)\).

  2. Distance-preserving: \(d_{\text{gauged}} \geq d_{\text{original}}\).

Formally: Given a valid gauging graph \(VG\), LDPC code property, incident bound, and original distance \(d_{\text{original}}\):

  1. (Sparsity) The three weight bounds from Theorem 21.6 hold.

  2. (Distance preservation) For all \(\texttt{logicalWeight} \geq d_{\text{original}}\), decoration weight, small subset \(U\), matter decrease \(\leq |U|\), and edge increase \(\geq |\partial U|\):

    \[ (\texttt{logicalWeight} + \texttt{decorationWeight}) - \texttt{matterDecrease} + \texttt{edgeIncrease} \geq d_{\text{original}} \]
Proof ▶

We prove both parts using a constructor.

Part 1: The sparsity result follows directly from GaugedCodeIsSparse applied with the given parameters.

Part 2: Let \(\texttt{logicalWeight}\), decoration weight, \(U\) (a small subset), matter decrease, and edge increase be given with the required bounds. The distance preservation follows directly from GaugedCodeDistancePreserved applied with these parameters.
